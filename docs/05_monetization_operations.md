# 収益化運用ガイド

## 1. KPI設定

### 主要KPI（毎日確認）

| KPI                           | 目標（初月） | 目標（3ヶ月後） | 計測方法                |
| ----------------------------- | ------------ | --------------- | ----------------------- |
| DAU（日次アクティブユーザー） | 50           | 500             | Play Console / Firebase |
| インストール数                | 300/月       | 1,000/月        | Play Console            |
| アンインストール率            | <30%         | <20%            | Play Console            |
| Premium転換率                 | 2%           | 5%              | 課金数/インストール     |
| ARPU（ユーザー単価）          | ¥10          | ¥30             | 総収益/DAU              |

### 収益KPI

| KPI                   | 計算式                         | 目標               |
| --------------------- | ------------------------------ | ------------------ |
| MRR（月間経常収益）   | Premium会員数 × ¥300           | ¥30,000（100人）   |
| 広告収益              | インプレッション × eCPM / 1000 | ¥10,000/月         |
| LTV（顧客生涯価値）   | 平均継続月数 × ¥300            | ¥1,800（6ヶ月）    |
| CAC（顧客獲得コスト） | 広告費 / 新規会員数            | ¥0（オーガニック） |

### 学習指標

| 指標              | 目標 | 意味                     |
| ----------------- | ---- | ------------------------ |
| セッション完了率  | >80% | 学習を最後まで完了       |
| 平均学習単語数/日 | 15語 | ユーザーエンゲージメント |
| 7日継続率         | >40% | 1週間後もアクティブ      |
| 30日継続率        | >20% | 1ヶ月後もアクティブ      |

## 2. 収益シミュレーション

### 月間収益モデル

```
【前提】
- インストール: 1,000/月
- DAU: 300
- Premium転換率: 3%

【収益計算】
サブスク収益:
  1,000 × 3% = 30人 × ¥300 = ¥9,000/月

広告収益:
  DAU 300 × 30日 = 9,000セッション
  バナー: 9,000 × ¥100 eCPM / 1000 = ¥900
  インタースティシャル: 3,000 × ¥500 eCPM / 1000 = ¥1,500
  広告計: ¥2,400/月

月間総収益: ¥11,400

【6ヶ月後の想定】
累計インストール: 6,000
Premium会員: 180人（継続率考慮で100人）
MRR: ¥30,000
広告: ¥8,000
月間収益: ¥38,000
```

## 3. 改善サイクル

### 週次レビュー（毎週月曜）

```
□ Play Console確認
  - インストール数推移
  - アンインストール率
  - クラッシュレポート
  - レビュー・評価

□ 収益確認
  - Play Console: サブスク収益
  - AdMob: 広告収益

□ 改善アクション決定
  - 優先度付けして1-2個に絞る
```

### 月次分析（毎月1日）

```
□ KPI達成率確認
  - 目標vs実績
  - 前月比較

□ ユーザーフィードバック分析
  - レビュー内容の傾向
  - 機能要望の集計

□ 競合分析
  - 類似アプリの動向
  - 新機能チェック

□ 次月の施策決定
```

## 7. 自動化・効率化

### 定期タスク自動化

```bash
# GitHub Actions でのリリース自動化例
# .github/workflows/release.yml

name: Release
on:
  push:
    tags:
      - 'v*'
jobs:
  build:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v3
      - name: Build AAB
        run: ./gradlew bundleRelease
      - name: Upload to Play Store
        uses: r0adkll/upload-google-play@v1
        with:
          serviceAccountJsonPlainText: ${{ secrets.SERVICE_ACCOUNT_JSON }}
          packageName: com.example.englishword
          releaseFiles: app/build/outputs/bundle/release/*.aab
          track: internal
```

## 9. 撤退基準

以下の場合は継続を再検討:

- 3ヶ月後に月間収益 < ¥5,000
- 継続率が著しく低い（7日 < 10%）
- 改善施策の効果が見られない

→ 別アプリへのピボットまたは学びを活かして次へ
